## プロジェクト概要

世界遺産「遠賀川水源地ポンプ室」（福岡県中間市）をブラウザ上の3D空間で再現する
「メタバース保存プロジェクト」のプロトタイプ。

- 寄付者の名前・メッセージをポンプ室のレンガに刻印し、訪問者が探して見られる
- 寄付金額に応じて企業広告・PR動画を空間内のモニターに掲載できる
- 将来的には cluster / VRChat 展開も視野（それらは Unity 等での別実装になる。
  本リポジトリはブラウザ版 = Three.js 実装）

## 実行方法

```
start.bat をダブルクリック
```

- ローカルHTTPサーバー（python -m http.server 8000）を起動し、ブラウザで
  `http://localhost:8000/index.html` を開く。
- **`file://` で index.html を直接開くと動画・画像テクスチャがブロックされるため、
  必ずHTTPサーバー経由で開くこと。**
- `.bat` ファイルには日本語（非ASCII文字）を書かない。cmd の文字コード（Shift-JIS）で
  化けてコマンドとして誤実行されるため、ASCIIのみで書く。

## アーキテクチャ（app.js）

外部3Dモデル・画像素材は使わず、テクスチャはすべて Canvas 2D で手続き生成している。
Three.js は r128（クラシックなグローバルビルド、CDN読み込み）。ES5 構文で書く。

### レンガ格子と刻印レジストリ（中核システム）

- レンガサイズは定数 `BRICK_W_M` / `BRICK_H_M` / `MORTAR_M`、解像度は `PX_PER_M`。
- `makeBrickWallTexture()` が壁テクスチャを生成しつつ、刻印したレンガを
  `brickRegistry`（key: `"wallId:row:col"` → donor オブジェクト）に登録する。
- donor は `type: "personal" | "company"`。personal は名前/日付/メッセージ、
  company は業種/従業員数/設立/URL などを持つ。`engrave` がレンガに描く短い表示名。
- `opts.featured` で特定位置に固定の刻印を置ける（現在 exemate を正面 y=1.6m に配置）。
- **描画パス順序が重要**: レンガ → 漆喰・汚れ → 蔦 → 刻印 → 窓・扉。
  刻印は必ずレンガ格子のマス内に収める（幅を測って自動縮小）。

### レンガ選択（クリックで情報表示）

- 毎フレーム、画面中央からレイキャスト → `wallBox`（BoxGeometry・面ごとにテクスチャ）に
  当たったら `face.materialIndex` で壁を特定し、`intersect.uv` をテクスチャの
  Canvas 座標に変換 → テクスチャ生成と同じ格子計算で row/col を割り出す。
- つまり **テクスチャ生成側とピッキング側で格子計算式を一致させる必要がある**。
  レンガサイズ・オフセット（千鳥配置: 奇数行は brickW/2 ずらし）を変えるときは両方直すこと。
- `wallFaceInfo` に各面の UV→ワールド方向（uDir）を実測定義済み。BoxGeometry の
  UV 向きに依存しているため Three.js のバージョンを変える場合は要再確認。
- ハイライト表示（金色=登録済み / 青白=空き）→ クリックで HTML ダイアログに
  拡大レンガ（Canvas 再描画）と登録情報を表示。

### 建物・環境

- 建物は双子切妻（M型屋根）。壁は1つの BoxGeometry に面別マテリアル、
  屋根斜面と妻壁三角は `makeQuadMesh` / `makeTriMesh` で手動生成。
- 実物写真準拠の要素: 非対称の妻面（左=丸窓 / 右=大アーチ窓）、隅の付柱＋石帽子、
  破風の歯飾り、避雷針マスト＋渡り線、越屋根、漆喰の剥がれ、正面右の赤レンガ附属屋、
  東側の貯水池（地面 ShapeGeometry に穴あけ＋護岸＋水面＋有刺鉄線フェンス）、
  黄手すり、鉄塔やぐら、県道73号相当の道路、周辺民家。
- 当たり判定は `colliders` 配列の AABB リスト。構造物を追加したらここにも追加する。

### 操作

WASD移動 / Shift走る / Space ジャンプ / ホイール=FOVズーム / 左クリック=レンガ選択 /
Q=ダイアログ閉じる / M=動画音声トグル / ESC=ポインターロック解除。

## 変更時の注意

- 動画の自動再生はブラウザポリシー上ミュート開始が必須。音声はユーザー操作（Mキー）で。
- テクスチャ Canvas サイズは `realW * PX_PER_M` で決まる。4096px を超えないよう注意
  （現在最大は東西壁 3960px）。
- 将来「実際の寄付者データ」を入れる場合は、`makeRandomDonor()` によるランダム生成を
  やめ、JSON/CSV から `brickRegistry` へ流し込む形にする想定。
