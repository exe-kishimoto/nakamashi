## プロジェクト概要

世界遺産「遠賀川水源地ポンプ室」（福岡県中間市）をブラウザ上の3D空間で再現する
「メタバース保存プロジェクト」のプロトタイプ。

- 寄付者の名前・メッセージをポンプ室のレンガに刻印し、訪問者が探して見られる
- 寄付金額に応じて企業広告・PR動画を空間内のモニターに掲載できる
- 将来的には cluster / VRChat 展開も視野（それらは Unity 等での別実装になる。
  本リポジトリはブラウザ版 = Three.js 実装）

## 実行方法

```
start.bat をダブルクリック
```

- ローカルHTTPサーバー（python -m http.server 8000）を起動し、ブラウザで
  `http://localhost:8000/index.html` を開く。
- **`file://` で index.html を直接開くと動画・画像テクスチャがブロックされるため、
  必ずHTTPサーバー経由で開くこと。**
- `.bat` ファイルには日本語（非ASCII文字）を書かない。cmd の文字コード（Shift-JIS）で
  化けてコマンドとして誤実行されるため、ASCIIのみで書く。

## ファイル構成

| ファイル | 内容 |
|---|---|
| `index.html` | エントリ。UI（開始オーバーレイ / HUD / レンガ刻印ダイアログ）と Three.js の CDN 読み込み |
| `app.js` | 本体。シーン構築・建物・周辺環境・操作系・レンガ選択システムすべて |
| `start.bat` | ローカルサーバー起動用（ASCIIのみ） |
| `asset/ad-movie-1.mp4` | 紹介動画（動画広告1）。※旧・日本語名からASCII名へリネーム |
| `asset/exemate.png` | スポンサー静止画（静止画広告1） |
| `asset/ad-movie-2.mp4`, `ad-movie-3.mp4` | 追加の動画広告（無ければ「準備中」表示） |
| `asset/ad-image-2.png`, `ad-image-3.png` | 追加の静止画広告（無ければ「準備中」表示） |

広告は**正面の前庭**に、中央通路を挟んで左（西）に動画3面・右（東）に静止画3面を配置
（通路を歩くと目に入る「広告アベニュー」）。ファイル名は `VIDEO_FILES` / `IMAGE_FILES` 配列で管理。素材が無い面は `makeAdPlaceholderTexture` のプレースホルダを表示し、
`loadeddata` / テクスチャ読込成功時に本素材へ差し替える。動画は自動再生のためミュート開始、
`M` キーで `setAdMuted` により全面まとめて音声トグル。操作ガイドは index.html の
`#controls-guide`（右下）で、ロック時に表示。

## アーキテクチャ（app.js）

外部3Dモデル・画像素材は使わず、テクスチャはすべて Canvas 2D で手続き生成している。
Three.js は r128（クラシックなグローバルビルド、CDN読み込み）。ES5 構文で書く。

### レンガ格子と刻印レジストリ（中核システム）

- レンガサイズは定数 `BRICK_W_M` / `BRICK_H_M` / `MORTAR_M`、解像度は `PX_PER_M`。
- `makeBrickWallTexture()` が壁テクスチャを生成しつつ、刻印したレンガを
  `brickRegistry`（key: `"wallId:row:col"` → donor オブジェクト）に登録する。
- donor は `type: "personal" | "company"`。personal は名前/日付/メッセージ、
  company は業種/従業員数/設立/URL などを持つ。`engrave` がレンガに描く短い表示名。
- `opts.featured` で特定位置に固定の刻印を置ける（現在 exemate を正面 y=1.6m に配置）。
- **描画パス順序が重要**: レンガ → 漆喰・汚れ → 蔦 → 刻印 → 窓・扉。
  刻印は必ずレンガ格子のマス内に収める（幅を測って自動縮小）。

### レンガ選択（クリックで情報表示）

- 毎フレーム、画面中央からレイキャスト → `wallBox`（BoxGeometry・面ごとにテクスチャ）に
  当たったら `face.materialIndex` で壁を特定し、`intersect.uv` をテクスチャの
  Canvas 座標に変換 → テクスチャ生成と同じ格子計算で row/col を割り出す。
- つまり **テクスチャ生成側とピッキング側で格子計算式を一致させる必要がある**。
  レンガサイズ・オフセット（千鳥配置: 奇数行は brickW/2 ずらし）を変えるときは両方直すこと。
- `wallFaceInfo` に各面の UV→ワールド方向（uDir）を実測定義済み。BoxGeometry の
  UV 向きに依存しているため Three.js のバージョンを変える場合は要再確認。
- ハイライト表示（金色=登録済み / 青白=空き）→ クリックで HTML ダイアログに
  拡大レンガ（Canvas 再描画）と登録情報を表示。

### 建物・環境

- 建物は双子切妻（M型屋根）。壁は1つの BoxGeometry に面別マテリアル、
  屋根斜面と妻壁三角は `makeQuadMesh` / `makeTriMesh` で手動生成。
- 妻面の窓は**正面から見て左＝丸窓 / 右＝大きな半円アーチ窓（`drawFanWindow`）**。
  BoxGeometry の front 面は +x が画面左に写るため、`gableFR`(+x)=丸窓 / `gableFL`(-x)=fan。
  正面下部は白漆喰主体（plaster 0.92）＋大きな貨物扉で、実物のシンプルな外観に合わせる。
- 実物写真準拠の要素: 非対称の妻面（左=丸窓 / 右=大アーチ窓）、隅の付柱＋石帽子、
  破風の歯飾り、避雷針マスト＋渡り線、越屋根、漆喰の剥がれ、正面右の赤レンガ附属屋、
  東側の貯水池（地面 ShapeGeometry に穴あけ＋護岸＋水面＋有刺鉄線フェンス）、
  黄手すり、鉄塔やぐら、道路、周辺民家。
- **実物の配置（航空写真準拠）**: 道路（県道73号）は正面妻壁のすぐ前を横切り、
  電柱・電線は道路沿い。貯水池は東の長辺のすぐ隣。敷地はフェンスで囲まれた
  コンパクトな区画（`SITE` 定数）で、正面に門がある。民家は道路の向かいと敷地外周。
- 当たり判定は `colliders` 配列の AABB リスト。構造物を追加したらここにも追加する。
- グラフィック: 影（PCFSoftShadowMap、`userData.shadow = "none"|"receive"` で除外指定し
  最後に一括 traverse で設定）、ACESトーンマッピング、レンガ壁は color + bump の
  2枚のCanvasを生成（`makeBrickWallTexture` が `{map, bump}` を返す）、
  空は雲と山並みを描いたドーム（SphereGeometry + BackSide）、建物足元にAOストリップ。

### 操作

WASD移動 / Shift走る / Space ジャンプ / ホイール=FOVズーム / 左クリック=レンガ選択 /
Q=ダイアログ閉じる / M=動画音声トグル / O=設定パネル / ESC=ポインターロック解除。
`O` はPC専用。ロック中はカーソルが無く⚙を押せないため、`settingsMode` を立てて
一時的に `controls.unlock()` する（この解除ではスタート用オーバーレイを出さない）。
パネルを閉じると `controls.lock()` で戻る。
`F` で浮遊モード（マイクラ風）: 重力なし・Space上昇/Shift下降/Ctrl加速。`updateMovement`
の先頭で `flyMode` を分岐（コライダー無視・上下自由、y は [0.6,70] にクランプ）。上のレンガの確認用。
看板の支柱は板の裏側(-z)へオフセットして前面へのめり込みを防いでいる。

### スマホ操作（`setupTouch`）

タッチ端末では `startMobile()` に入り、ポインターロックの代わりに `lookEuler` を直接回す。
右下スティック=移動 / 全画面 `#look-layer` のドラッグ=視点 / ピンチ=FOV / タップ=レンガ選択。
左下=ジャンプ・走る・浮遊、左上=🔇（音声トグルだけは配置変更の対象外）。
⚙ボタンの位置は `placeSettingsBtn()` がタイトルバッジの実測高から決める（固定値だと機種で重なる）。
設定パネルは画面高から `max-height` を算出してスクロール（`touch-action` を `none` にすると
指でスクロールできなくなるので `pan-y`）。横向きは CSS 高さが 300px 程度しかないため上端から配置。
コントローラー配置はプリセット/ドラッグで変更でき localStorage (`nakamashi.layout.v1`) に保存。
- **`resize` イベントは使わない**。Android は回転時に *回転前の* サイズで resize を投げることがあり、
  横向きなのに縦向きの高さでUIを組んでしまう。`watchViewport()` が毎フレーム実寸
  （`visualViewport.height` 優先）を見て、変化した時だけカメラ・レンダラ・`reflowUI()` を更新する。
  画面サイズに依存するUIを足す時は `reflowUI` に載せること。

### ジェスチャー（`setupGesture`・顔の向きで視点移動）

⚙→「😊 顔で視点移動」で MediaPipe Face Landmarker を CDN から**遅延**読込（有効時のみ約3MB）。
ES5 のクラシックスクリプトから import できないため、module スクリプトを動的に差し込んで
`window.__mpVision` 経由で受け取る。`outputFacialTransformationMatrixes` の 4x4（列優先）の
第2列＝顔の正面ベクトルから yaw/pitch を取る。
**顔の向き＝視点の向き（絶対）にすると画面から目が離れて酔うため、傾け“量”→回転“速度”**
（`DEAD` の遊びを超えた分を二乗カーブで 0→1）。開始時に3秒カウントで正面を基準化（`calibrate`）。
カメラ映像の鏡像有無は端末差があるので、左右が逆なら⚙の「左右の向きを反転」で切替（`invert`）。

## 変更時の注意

- 動画の自動再生はブラウザポリシー上ミュート開始が必須。音声はユーザー操作（Mキー）で。
- テクスチャ Canvas サイズは `realW * PX_PER_M` で決まる。4096px を超えないよう注意
  （現在最大は東西壁 3960px）。
- 将来「実際の寄付者データ」を入れる場合は、`makeRandomDonor()` によるランダム生成を
  やめ、JSON/CSV から `brickRegistry` へ流し込む形にする想定。
